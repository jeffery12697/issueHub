## Design Context

### Users
Software developers and cross-functional teams (devs, designers, PMs) tracking tasks, bugs, and sprints. Long sessions at a desk; need fast navigation, clear hierarchy, and zero cognitive overhead. The interface competes for attention against code editors and terminals — it must earn every glance and respond instantly. Inspired by Linear, ClickUp, and Jira.

### Brand Personality
**Focused · Fast · Clean** — a productivity tool that gets out of the way. The interface should feel confident and sharp, not flashy. Users should feel in control. Think "tool" not "product."

---

### Aesthetic Direction

- **Visual tone**: Minimal and sharp, leaning Linear — tight spacing, white/`slate-50` surfaces, violet as the single accent
- **Primary accent**: `violet-600` for interactive/active/actionable states; `violet-700` on hover; `violet-50`/`violet-100` for tinted backgrounds; `violet-300` for accent borders
- **Neutrals**: Slate palette exclusively — `slate-50` pages, `white` cards, `slate-100` subtle fills, `slate-200` borders, `slate-300` dividers, `slate-400/500` muted text, `slate-700/800/900` primary text
- **Typography**: System-ui (confirmed choice) — native sans-serif on each OS (SF Pro on macOS, Segoe UI on Windows). Fast, zero FOUT, contextually appropriate for a developer tool
- **Radius**: `rounded-xl` for cards and panels; `rounded-lg` for buttons, inputs, dropdowns; `rounded-full` for badges, pills, avatars
- **Shadows**: Minimal — `shadow-sm` only. Elevation expressed through borders (`border-slate-200`), not heavy drop shadows. `shadow-md` on hover states only (e.g., kanban cards)
- **Dark mode**: **Yes, planned as a priority.** Avoid hardcoded light-only values in new components. Prefer CSS-variable-friendly patterns; add `dark:` Tailwind variants when building new surfaces
- **Anti-references**: Generic SaaS purple gradients, nested card-in-card patterns, glassmorphism, hero metric dashboards, rainbow icon palettes, bounce/elastic animations, gradient text

---

### Design Principles

1. **Clarity over decoration** — every visual element must serve information hierarchy; remove anything that doesn't help the user make a decision
2. **Violet is a signal, not wallpaper** — use `violet-600` only for interactive, active, or actionable states; never as a background decoration or identity color
3. **Density with breathing room** — task-tracker users want information density; use compact rows and tight spacing at the row level, but preserve whitespace at card and section boundaries
4. **Consistent motion** — transitions are Tailwind-default 150ms, `ease` easing only; no bouncing, no elastic, no delayed entrance animations; always respect `prefers-reduced-motion` (global CSS override in `index.css`)
5. **Status speaks first** — priority colors (sky/amber/orange/red) and status badges are the highest-priority visual signals on any task surface; keep them legible and never desaturate them

### Icon System

**Transitioning to Lucide React** (`lucide-react` package) — the maintained Feather Icons fork.

Until migration is complete, new inline SVGs follow the same visual style:
```
fill="none" stroke="currentColor" strokeWidth="2"
strokeLinecap="round" strokeLinejoin="round"
```

Icon sizes:
- Inline in button/link: `w-[13px] h-[13px]` (`width="13" height="13"`)
- Standalone header/action: `w-[18px] h-[18px]` (`width="18" height="18"`)
- Empty state illustration: `w-6 h-6` or `w-8 h-8`

All decorative icons: `aria-hidden="true"`. Interactive icon-only buttons: `aria-label="…"` on the button.

No emoji icons in interactive elements (emoji announce literally in screen readers and render inconsistently across OS).

### Responsive Strategy

**Desktop-primary** (1024px+). Mobile must not break; tablet should feel comfortable.

Key breakpoints: `sm` (640px) for most adaptations, `md` (768px) for column visibility.

- **Headers with nav** (WorkspaceHeader): two-row layout on mobile — breadcrumb row + scrollable nav row below
- **Action toolbars**: `flex-col sm:flex-row` stacking on narrow screens; action groups use `flex-wrap`
- **Data tables**: `overflow-x-auto` inner wrapper + `min-w-[640px]` on `<table>` to trigger scroll before collapse
- **Header list names**: `truncate max-w-[120px] sm:max-w-none`
- **Search input**: `w-32 sm:w-48`; dropdown `w-[min(24rem,calc(100vw-1rem))]`
- **User name in header**: `hidden sm:inline`
- **Stat card grids**: `grid-cols-1 sm:grid-cols-2`

---

### Motion Guidelines

- **Duration**: Tailwind default (150ms) for `transition-colors`; 200ms max for layout/transform
- **Easing**: `ease` (ease-in-out equivalent) only — no bounce, no elastic, no spring
- **Allowed**: color transitions, opacity, `translate-y` lift on hover (`-translate-y-0.5`)
- **Forbidden**: entrance animations, staggered list reveals, spinning loaders (use skeleton instead), layout-animating transitions (`width`, `height`, `margin`)
- **Reduced motion**: Global CSS override in `frontend/src/index.css` disables all transitions/animations when `prefers-reduced-motion: reduce`
- **Loading states**: Always use skeleton placeholders (`animate-pulse bg-slate-200`), never spinners or plain text

---

### Delight Philosophy

**Subtle micro-interactions only** — the kind a user notices unconsciously, not explicitly. Never decorative.

Appropriate delight:
- Checkbox state transitions (gentle scale/color change on complete)
- Status badge color shift when a task moves to "done"
- Kanban card lift (`hover:-translate-y-0.5`) for drag affordance
- Focus ring appearance (`ring-2 ring-violet-500`) as interaction confirmation

Not appropriate:
- Confetti, particle effects, celebration overlays
- Bouncing or elastic animations
- Animated illustrations or mascots
- Attention-seeking micro-copy ("Amazing! You did it! 🎉")

---

### Accessibility Baseline

- WCAG AA as minimum for all text contrast
- Focus ring: `focus:outline-none focus:ring-2 focus:ring-violet-500` on all interactive elements
- Dialogs/dropdowns: `role="dialog"` + `aria-label`
- Disclosure buttons: `aria-expanded={bool}`
- Icon-only buttons: `aria-label="…"` on the button element
- Combobox/search: `role="combobox"` + `aria-autocomplete` + `aria-expanded` + `aria-controls`
- Screen reader nav: keyboard-navigable search results (`ArrowUp/Down/Enter`)
- All SVG icons in interactive contexts: `aria-hidden="true"` (label lives on parent button)
